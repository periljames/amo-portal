# UI Regression Checklist (AppShell V2)

## Pages to review
- Dashboard
- Document Control
- Audits & Inspections
- Change Control
- CAR Register
- Training & Competence
- Quality Events
- KPIs & Review

## Visual checks
- Sidebar remains visible on desktop; active item matches the current route.
- Top header stays sticky, shows status chips, and does not overlap content.
- Trial status appears as a header chip (not a full-width banner).
- System errors appear as toasts; widget errors are inline.
- No duplicated module navigation tiles appear inside QMS pages.
- Page headers use consistent spacing and typography.
- Empty states are compact and action-oriented.

## Interaction checks
- Sidebar QMS sub-navigation highlights active module.
- Trial chip menu actions (View plans, Convert, Hide 7 days, Hide session) work.
- Create schedule drawer opens/closes and submits successfully.
- Audit list filters and search update results.
